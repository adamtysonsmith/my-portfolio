Jade2HTML

Allows you to develop a website using jade templating with any data source, and compile to static HTML for fast performance and easy deployment.

Node.js Express Jade

Sometimes you just want a website that has all the benefits of static HTML, without actually writing 100,000 pages of HTML. Well, maybe you don't, but I do! Static HTML sites are blazing fast, they can be hosted anywhere, and they do not require a database, so the security concerns are minimal. On the other hand, huge static HTML websites are a pain in the ass to maintain, and accumulate tons of repetitive code, yuck. If only there was a solution..

These were the thoughts that inspired me to write this tool. This is more of a development workflow that uses Node, Express, and any JSON data source that you like. The only requirement is that each data object contains a 'filename' property that can be used for routing during development. It will use that unique filename to write the html files when you compile. Easy as pie.

Check out the documentation on Github, I have provided some functions to let you build out the directory structure, pass in data, and give you fine grained control when compiling your templates.

Goodwifi

Life is too short for poor quality wifi, find the good wifi near you!

Ionic AngularJS Raspberry Pi D3.js Node.js

The idea for Goodwifi came from my good friend Alex Moore. He identified a real problem that most of us deal with at some point - You want to sit down at a cafe or restuarant to get some work done, and the wifi sucks!

We decided to run Node.js on a Raspberry Pi, which connects to the cafe's wifi network. We wrote a simple application that tests the network speeds on the wifi, and sends the data hourly to a MongoDB instance in the cloud.

In order for consumers to check the wifi at their favorite locations, we wrote an iOS app using the Ionic Framework. The app allows you to browse the locations (that are using the Goodwifi hardware) and view the networks speeds using some D3 charts. We have some additional metrics that we collect in order to calculate a "Wifi Quality Score" which is also displayed and updated each hour. This should allow you to see the history of the wifi performance over time, or which times of day are optimal for wifi use. You can save your favorite locations to pull up anytime you want to check the wifi.

This project is currently in the prototyping phase, but we hope to work out the business details and start testing wifi locations in Boulder by the end of the year.
